## Git 版本控制練習說明

### 1. Git Revert 紀錄
我在開發過程中進行了一次 `git revert` 練習。
* **情境**：沒發現footer打錯字，還push上去
* **操作**：我沒有直接刪除程式碼，而是使用 `git revert` 指令來產生一個反向的 Commit，這樣可以保留修改紀錄但復原效果。

### 2. A/B Test 實驗 (Skills 區塊)
針對「技能展示 (Skills)」區塊，我實作了兩種不同的視覺呈現版本，並發起了兩個 Pull Request：

* **版本 A (feature/skills-bars)**：改footer
* **版本 B (feature/skills-tags)**：改顏色

**最終決定：採用版本 A (進度條)**
* **採用原因**：改成正確的footer
* **未採用原因 (版本 B)**:顏色不好看

## 技術亮點
* **HTML5 / CSS3**：程式碼寫得很乾淨，還用變數統一管理顏色，以後要改版超方便。
* **RWD 設計**：不管是用手機還是電腦看，版面都會自動調整，怎麼看都不會跑版。
* **互動效果**：點選單畫面會滑順捲動，滑鼠移過去卡片還會浮起來，操作起來不無聊。

## 心得
這是我人生第一次做網頁。如果沒有AI輔助，我根本寫不出這些HTML和CSS。我只要我要把Hello Kitty結合搖滾的風格告訴AI，它幫我產出程式碼，我再試著去理解怎麼把檔案跑起來。這次作業最難的其實是Git操作，像A/BTest和Revert這種版本控制，邏輯對新手來說很抽象，還好有AI一步步帶我操作指令。這讓我感覺寫程式不再那麼可怕，我更像是在「指揮」AI完成任務，雖然基礎還不穩，但能從零到有弄出一個公開網站，真的蠻驚訝自己能做到的。
